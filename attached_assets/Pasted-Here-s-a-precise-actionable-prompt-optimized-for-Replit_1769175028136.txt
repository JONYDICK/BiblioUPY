# SECURE CAMPUS LIBRARY PLATFORM - REPLIT STARTUP SIMULATION
**Project Context:** University course simulating a startup building secure software for a polytechnic institution's digital library. Students upload/share academic papers/books under strict security protocols. All code must demonstrate security-by-design principles.

## CORE REQUIREMENTS
### 1. USER SYSTEM (OWASP A1:2021)
- Implement OAuth2/SAML integration with university SSO (mock university domain: `@politecnica.edu`)
- Enforce 2FA for admin/moderator roles using TOTP (Python: `pyotp`)
- RBAC Matrix:
  * Students: Upload documents (max 50MB), comment (sanitized), create reading lists
  * Professors: Validate user uploads, tag resources, moderate comments
  * Security Team (student role): View audit logs, run vulnerability scans
- GDPR-compliant profiles: Hide emails by default, data deletion endpoint (`/api/user/anonymize`)

### 2. SECURE CONTENT PIPELINE (OWASP A5:2021)
- Document uploads:
  * Block executable files (`.exe`, `.sh`, `.bat`)
  * Auto-scan with ClamAV Docker container (integrate via socket)
  * Strip EXIF data from PDFs/images using `exiftool`
- Version control: Track paper revisions with Git LFS simulation (store hashes in PostgreSQL `document_versions` table)

### 3. SECURITY-FIRST COLLABORATION
- Annotation system: 
  * Sanitize all user input with `bleach` (allow only `<b>`, `<i>`, `<a>`)
  * Rate limiting: 5 comments/minute/user (Redis-backed)
- Sharing controls:
  * Generate expiring share links (JWT with 24h TTL)
  * Default visibility: "University members only"

### 4. TECHNICAL SECURITY HARDENING
- **MUST implement in first PR:**
  ```python
  # Example security headers middleware
  @app.after_request
  def add_security_headers(response):
      response.headers['Content-Security-Policy'] = "default-src 'self'; script-src 'nonce-{random}'"
      response.headers['Strict-Transport-Security'] = 'max-age=31536000; includeSubDomains'
      return response
  ```
- Secrets management: Use Replit Secrets for DB credentials (never hardcode)
- Dependency scanning: Add `safety` to pre-commit hook (`safety check --full-report`)
- Audit trail: Log all file uploads/deletions to `/var/log/audit.log` (structured JSON)

### 5. STARTUP METHODOLOGY INTEGRATION
- SDLC artifacts in `/docs`:
  * `THREAT_MODEL.md`: STRIDE analysis for document upload flow
  * `SECURITY_DECISIONS.md`: Justify framework choices (e.g., "Django over Flask for built-in CSRF protection")
- Bug bounty simulation:
  * `/report-vulnerability` endpoint with form
  * Auto-issue "security credits" for valid reports (stored in `user_credits` table)

### 6. LEGAL & ETHICAL SAFEGUARDS
- DMCA workflow:
  1. User report button on each document
  2. Auto-quarantine reported content (move to `/quarantine` storage)
  3. Admin review panel with evidence display
- License selector during upload: Mandatory CC license dropdown (CC-BY-NC-SA 4.0 default)

### 7. EDUCATIONAL COMPONENTS
- Security dashboard (`/professor/security-dashboard`):
  * Live metrics: "Vulnerabilities patched", "Pending moderation queue"
  * "Security Health Score" calculated from: 
    `(patched_vulns * 0.6) + (tested_endpoints * 0.4)`
- Interactive tutorial: 
  * `/lab/xss-simulator` - Safe sandbox to practice payload injection

### 8. INFRASTRUCTURE (Replit-Optimized)
- Database: PostgreSQL (Replit DB) with row-level security
- File storage: Replit Blob storage with encryption-at-rest
- CI/CD pipeline (`.replit` config):
  ```toml
  [deploy]
  run = ["pytest --cov=app tests/", "bandit -r app/"]
  failure_exit_code = 1
  ```

### 9. UX SECURITY PATTERNS
- Error messages: NEVER reveal stack traces (generic "Request failed [ERR-SEC-403]")
- UI indicators:
  * Padlock icon (green/red) showing current session security status
  * Mandatory "Security Checklist" modal on first document upload

### 10. METRICS & TRANSPARENCY
- Public security page (`/transparency`):
  * Last dependency scan date
  * Critical vulnerabilities resolved (30-day count)
  * Uptime status (integrate with Replit status API)

## DELIVERABLES FOR COURSE
1. Working prototype in Replit with all security headers implemented
2. `/docs/SECURITY_CERTIFICATION.md` containing:
   - Screenshots of successful ZAP scans
